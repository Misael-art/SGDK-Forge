# 05 - Guia de Edicao

Ordem sugerida para evoluir o projeto:

1. renomear o projeto e atualizar `.mddev/project.json`;
2. ajustar `doc/01-visao-geral.md`;
3. adicionar novas cenas em `src/scenes/`;
4. criar recursos em `res/data/`;
5. declarar recursos reais quando necessario;
6. validar sempre via wrapper.

Checklist rapido:
- o build continua limpo?
- a documentacao ainda descreve a verdade?
- o estado global continua pequeno?
- o projeto continua copiavel?
